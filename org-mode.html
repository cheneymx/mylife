1 Tips

  1. 如果网页中有无效链接, Org 会停止生成 html 文件. 有些时候, 我们不能将所有的链接都补齐, 这种情况, 可以使用如下设置, 让 org 忽略无效连接. 

    #+OPTIONS: broken-links:t

  2. Org 发布整个项目的时候, 他不会去查看网页的情况, 而是使用 ~/.org-timestamps 记录上次发布的信息. 只要源文件在上次发布 之后没有修改, 则项目发布时, 该文件就不会被重新 publish 为 html, 即使相关 html 文件已经删除. 可以使用 C-u 来强制重新 发布所有文件 

    C-u M-x org-publish-project

  3. Org 发布 html 会将 ^ 和 _ 后的当做上标和下标, 很多时候非常多余. 在 Customize 下找 Option Org Export With Sub Superscripts. 这里有个非常好的选项, 就是仅仅在 ^ 和 _ 后部分文字用 {} 括起来才会启动上下标. 这就非常好了. 设置保存 后, 在 .sys_custom.el 里会多出如下的一条设置: 

    '(org-export-with-sub-superscripts '{})

    另外, 对于单个文件取消上下角标可以使用如下 option 

    #+OPTIONS: ^:nil
